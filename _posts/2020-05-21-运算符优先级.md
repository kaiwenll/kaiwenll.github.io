### Java 运算符优先级

| 运算符               | 优先级                                   |
| :------------------- | :--------------------------------------- |
| postfix              | `*expr*++ *expr*--`                      |
| unary                | `++*expr* --*expr* +*expr* -*expr* ~ !`  |
| multiplicative       | `* / %`                                  |
| additive             | `+ -`                                    |
| shift                | `<< >> >>>`                              |
| relational           | `< > <= >= instanceof`                   |
| equality             | `== !=`                                  |
| bitwise AND          | `&`                                      |
| bitwise exclusive OR | `^`                                      |
| bitwise inclusive OR | `|`                                      |
| logical AND          | `&&`                                     |
| logical OR           | `||`                                     |
| ternary              | `? :`                                    |
| assignment           | `= += -= *= /= %= &= ^= |= <<= >>= >>>=` |

```
这个表给出Python的运算符优先级（从低到高）.

从最低的优先级（最松散地结合）到最高的优先级（最紧密地结合）。

这意味着在一个表达式中，Python会首先计算表中较下面的运算符，然后在计算列在表上部的运算符。
```
